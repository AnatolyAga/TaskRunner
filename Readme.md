# GIT

**Git - это распределённая система контроля версий, разработанная
Линусом Торвальдсом в 2005 году для управления исходным кодом ядра
Linux. Сегодня Git является стандартом в индустрии разработки
программного обеспечения и незаменимым инструментом для любого
разработчика**

## Основные функции и особенности: 
**Контроль версий:** Сохраняет историю всех изменений проекта, позволяя посмотреть, кто, когда и что изменил, и вернуться к любой точке в истории.

**Распределенная архитектура**: Каждый разработчик имеет полную копию репозитория (хранилища кода и его истории) на своем компьютере, в отличие от централизованных систем.

**Совместная работа:** Упрощает работу в команде благодаря ветвлению (создание параллельных веток для новых функций) и слиянию (объединение изменений).

**Локальная работа:** Большинство операций выполняется локально, без необходимости постоянного подключения к серверу.

**Не только код:** Используется для документации, статей, баз знаний и других типов файлов. 

**Git** — это сама программа (утилита командной строки) для контроля версий, которую нужно установить.
GitHub, GitLab и Bitbucket — это веб-сервисы, которые предоставляют удаленные хранилища для ваших Git-репозиториев и добавляют социальные функции и инструменты для совместной работы, но не являются самим Git. 

## Основные команды Git:

**Команды Git вводятся в терминале:**

```
git init — инициализация локального репозитория

git status — получение информации о текущем состоянии репозитория

git add — добавление файлов к следующему коммиту

git commit -m "message" — создание коммита

git log — вывод истории коммитов с их хеш-кодами

```
